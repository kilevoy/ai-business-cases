# Автоматизация поиска климатических данных по СП

> Статический PWA-атлас для быстрого поиска нормативных климатических параметров по населенным пунктам РФ: снег, ветер, гололед, сейсмика, промерзание грунта и температуры по СП.

---

## Business Problem

При подготовке инженерных расчетов и коммерческих предложений нужно быстро находить климатические параметры для объекта: снеговой район, ветровой район, гололедную нагрузку, сейсмичность, глубину промерзания и расчетные температуры. В ручном процессе данные приходится собирать из разных нормативных документов и справочников:

- СП 20.13330 для снеговых, ветровых и гололедных нагрузок;
- СП 14.13330 для сейсмичности;
- СП 131.13330 для климатических температур и периодов;
- СП 22.13330 для глубины сезонного промерзания;
- внешние сверочные источники и инженерные таблицы.

Такой поиск занимает время, плохо масштабируется на поток заявок и повышает риск ошибочного выбора города, района или численного значения нагрузки.

---

## What Was Done

Создан автономный веб-атлас климатических данных. Пользователь вводит город или регион, выбирает населенный пункт и получает структурированную карточку с расчетными параметрами, источниками и статусами проверки.

| Feature | Description |
|---------|-------------|
| Поиск населенного пункта | Поиск по названию города или региона с подсказками |
| Единый JSON-справочник | `settlements-climate.json` содержит 1096 записей по населенным пунктам |
| Нагрузки по СП 20 | Снеговой район и Sg, ветровой район и w0, гололедный район и толщина гололеда |
| Сейсмика | Баллы сейсмичности по СП 14.13330 / ОСР-2015 |
| Промерзание | Глубина сезонного промерзания по типам грунтов |
| Климатические периоды | Холодный и теплый периоды по СП 131.13330 |
| Температурный график | Среднемесячные температуры и среднегодовая температура |
| Статусы данных | Для записей отображаются статусы `verified` / `partial` |
| PWA-режим | Manifest, иконки и service worker для установки и офлайн-кэша |
| Светлая/темная тема | Выбор темы сохраняется в LocalStorage |

## Process Automated

Автоматизирован участок подбора нормативных климатических данных для расчетов:

1. Инженер или менеджер вводит название города.
2. Система показывает подходящие населенные пункты и регионы.
3. После выбора города открывается карточка с нормативными параметрами.
4. Данные сгруппированы по расчетным разделам: нагрузки, холодный период, теплый период, промерзание, температурный график.
5. Для каждого ключевого значения отображается источник и статус проверки.
6. Полученные параметры можно использовать в расчетах металлоконструкций, ограждающих конструкций и предварительной подготовке КП.

## Architecture / Modules

| Module | Role |
|--------|------|
| `index.html` | Весь интерфейс приложения, поиск, вкладки, карточки параметров и график температур |
| `settlements-climate.json` | Основной справочник климатических данных по 1096 населенным пунктам |
| `manifest.json` | PWA-метаданные: название, иконки, standalone-режим |
| `sw.js` | Service worker для кэширования HTML, JSON, manifest и иконок |
| `columns-glossary.md` | Описание структуры исходной мастер-таблицы и колонок |
| `json-glossary.md` | Описание JSON-схемы, статусов и полей климатических данных |

Проект не требует сборки и серверной части: это статический HTML/JS/CSS-инструмент, который загружает локальный JSON-справочник.

## Business Impact

- **Сокращение времени поиска данных**: вместо ручного просмотра нескольких СП пользователь получает карточку по городу в одном интерфейсе.
- **Снижение риска ошибок**: снег, ветер, гололед, сейсмика и климатические параметры привязаны к одной записи населенного пункта.
- **Стандартизация расчетной базы**: одинаковые справочные данные можно использовать в разных инженерных калькуляторах.
- **Прозрачность источников**: у параметров сохраняются ссылки на нормативные источники и статусы проверки.
- **Автономность**: атлас работает как статическая PWA-страница и может открываться без серверной инфраструктуры.
