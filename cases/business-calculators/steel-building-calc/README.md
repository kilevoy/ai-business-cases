# Автоматизация подбора стального каркаса промышленных зданий

> Веб-инструмент в процессе разработки для переноса инженерных Excel-расчетов стального каркаса в единое React/TypeScript-приложение: колонны, фермы, прогоны, балки покрытия, подкрановые балки и оконные ригели.

---

## Business Problem

Подготовка расчетной базы для коммерческого предложения по промышленному зданию требует связать несколько инженерных блоков: геометрию здания, климатические нагрузки, подбор колонн, прогонов, ферм, балок и отдельных узлов. В Excel-процессе это создает несколько рисков:

- параметры здания приходится переносить между разными расчетными листами;
- изменения пролета, длины, высоты, шага рам или снегового района требуют пересчета связанных модулей;
- часть справочников и формул сложно контролировать при развитии проекта;
- итоговая масса и стоимость зависят от корректности ручной сборки результатов;
- регрессионная проверка изменений в формулах затруднена.

Цель проекта — постепенно перевести Excel-расчеты в единый клиентский инструмент, где расчетные модули можно проверять тестами и использовать как основу для подготовки КП.

| Прогоны | Отдельный модуль расчета прогонов с подбором вариантов и проверкой нагрузок |

| Балочная клетка | Модуль для расчета второстепенных и главных балок, а также колонн балочной клетки |

| Оконные ригели | Отдельный расчетный модуль с данными, перенесенными из workbook |

| Проверочные скрипты | Скрипты для сверки с Excel-логикой и проверки отдельных расчетных модулей |

## Process Automated

Автоматизируется процесс предварительного инженерно-коммерческого подбора стального каркаса:

1. Пользователь задает параметры здания: пролет, длину, высоту, шаг рам, район строительства, покрытие и нагрузки.
2. Система рассчитывает климатические и производные параметры.
3. Модуль колонн подбирает подходящие профили и показывает расчетные проверки.
4. Связанные модули считают прогоны, фермы, балки, подкрановые элементы и оконные ригели.
5. Общая логика здания передает результаты между разделами, чтобы снизить ручной перенос данных.
6. Проверочные скрипты помогают сравнивать результаты с исходными Excel-сценариями.

## Architecture / Modules

| Module | Role |
|--------|------|
| `src/App.tsx` | Основной интерфейс приложения и переключение расчетных вкладок |
| `src/building` | Общий контекст здания, синхронизация параметров и передача результатов между модулями |
| `src/calc/engine.ts` | Основной расчет подбора стальных колонн |
| `src/calc/wind.ts` | Расчет ветровых коэффициентов по СП 20.13330 |
| `src/calc/purlin` | Расчет прогонов и подбор вариантов |
| `src/calc/truss` | Расчет фермы и проверка элементов |
| `src/calc/beamCell` | Расчет балочной клетки |
| `src/calc/craneBeam` | Расчет подкрановой балки на основе workbook-generated данных |
| `src/calc/windowRiegel` | Расчет оконных ригелей на основе workbook-generated данных |
| `scripts` | Сверка расчетов, Excel-oracle сценарии и вспомогательные проверки |

## Business Impact

- **Снижение ручного переноса данных**: общий контекст здания помогает использовать одни параметры в нескольких расчетных разделах.
- **Постепенная стандартизация расчетов**: отдельные Excel-формулы переводятся в TypeScript-модули с тестами.
- **Быстрая проверка вариантов**: подбор профилей можно выполнять в интерфейсе без ручного перебора справочников.
- **Контроль качества разработки**: расчетные изменения можно проверять через unit-тесты и scripts-сверки.
- **Основа для единого КП-процесса**: проект формирует базу для будущей сводной массы, стоимости и спецификации по каркасу.
