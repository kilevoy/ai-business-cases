# Автоматизация обновления и сравнения прайсов Металл Профиль

> Локальный MVP для обработки PDF-прайсов поставщика, извлечения цен по ключевым товарным группам и публикации статической страницы с историей прайсов.

---

## Business Problem

Коммерческие и расчетные инструменты зависят от актуальных цен поставщика. Если прайсы приходят в PDF, менеджеру или инженеру приходится вручную искать нужные страницы, переносить цены в таблицы, проверять дату действия прайса и сравнивать изменения с предыдущей версией.

Такой процесс создает несколько рисков:

- цены могут быть перенесены с ошибкой;
- новый прайс может не попасть в расчетные инструменты вовремя;
- сложно быстро понять, что изменилось относительно предыдущей версии;
- одинаковые данные приходится вручную обновлять в нескольких местах;
- при изменении структуры PDF ручная проверка занимает дополнительное время.

---

## What Was Done

Создан локальный инструмент, который берет PDF-прайсы из папки `input/`, извлекает структурированные данные и собирает статическую страницу для просмотра цен.

| Feature | Description |
|---------|-------------|
| Обработка PDF | Скрипты читают прайсы Металл Профиль из `input/` |
| История прайсов | Даты прайсов извлекаются из имени файла и сортируются по времени |
| Сэндвич-панели | Парсинг цен по классам 1 и 2, толщине, типам панелей и стандартным цветам |
| Фасонные изделия | Расчет цен для фасонных изделий по формуле `цена плоского листа × 1,9` |
| Профилированный и плоский лист | Парсинг раздела с полимерным покрытием и фильтрами по классу, покрытию и толщине |
| Сравнение версий | Для сэндвич-панелей формируется сравнение с предыдущим прайсом |
| Локальная загрузка | Есть локальный сервер для загрузки PDF через интерфейс и запуска обработки |
| Статическая публикация | Готовые HTML и JSON публикуются из папки `site/` на GitHub Pages |

---

## Process Automated

Автоматизирован процесс обновления прайсов для дальнейших коммерческих и инженерных расчетов:

1. Пользователь кладет новый PDF-прайс в папку `input/` или загружает его через локальный интерфейс.
2. Скрипт определяет дату прайса по имени файла.
3. Парсер находит нужные страницы PDF по ключевым словам, а не только по жесткому номеру страницы.
4. Цены и справочные блоки извлекаются в JSON.
5. Для опубликованного сайта пересобираются HTML-страницы и JSON-данные.
6. Пользователь видит актуальный прайс, историю дат и может сравнивать версии.
7. Обновленные данные можно использовать в расчетных инструментах и коммерческой работе.

## Architecture / Modules

| Module | Role |
|--------|------|
| `input/` | Входные PDF-прайсы поставщика |
| `scripts/update_sandwich_panels.py` | Парсинг сэндвич-панелей, стандартных цветов и фасонных изделий |
| `scripts/build_profiled_sheet_page4.py` | Парсинг раздела профилированного и плоского листа |
| `scripts/local_upload_server.py` | Локальный HTTP-сервер для загрузки PDF и запуска обоих парсеров |
| `data/` | Структурированные JSON-данные после обработки |
| `site/` | Готовые HTML/JSON-файлы для GitHub Pages |
| `scan_and_update_input.bat` | Сканирование папки с PDF и запуск обновления |
| `run_local_uploader.bat` | Запуск локального загрузчика прайсов |

## Business Impact

- **Снижение ручных ошибок**: цены извлекаются из PDF по единому алгоритму.
- **Быстрое обновление расчетной базы**: новый прайс можно добавить через папку или локальную загрузку.
- **Прозрачная история изменений**: сайт показывает сохраненные даты прайсов и текущую версию.
- **Повторяемый процесс**: обновление выполняется одинаково для каждого нового PDF.
- **Подготовка данных для калькуляторов**: JSON можно использовать как источник цен в других расчетных инструментах.
