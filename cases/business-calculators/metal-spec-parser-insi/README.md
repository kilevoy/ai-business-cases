# КМД → ведомость металла ИНСИ

> AI-assisted MVP в разработке для автоматизации разбора КМ/КМД-документации: PDF/Excel → выборка металла → нормализация профилей → сопоставление с номенклатурой ИНСИ → Excel для снабжения.

## Business Problem

Проектная документация КМ/КМД часто приходит в PDF или Excel, где спецификации, выборки металла, приложения и листы профилей разбросаны по разным страницам и листам. Для подготовки ведомости в корпоративной форме ИНСИ специалисту приходится вручную:

- находить нужные таблицы среди сотен страниц;
- переносить профили, марки стали, длины, количества и массы;
- приводить разные написания профилей к единому виду;
- сопоставлять позиции с внутренней номенклатурой;
- проверять спорные материалы и несопоставленные строки;
- формировать Excel для снабжения или производства.

Основной риск процесса — не только скорость, но и ошибки: пропущенная строка, неверный материал, подмена профиля похожим аналогом или неконтролируемая масса.

## What Is Being Built

Создается web-сервис, в котором пользователь загружает проектную документацию и номенклатуру ИНСИ. Система локально разбирает документы, показывает найденные таблицы, извлекает строки металла, классифицирует материалы, нормализует профили и формирует Excel-выгрузку.

| Feature | Description |
|---------|-------------|
| Загрузка документов | PDF КМ/КМД или Excel-ведомость проекта, отдельный Excel номенклатуры ИНСИ |
| Поиск таблиц | Локальный поиск спецификаций, выборок металла, листов профилей, приложений А/Б и деталей отправочных марок |
| Классификация таблиц | Фильтры по типу таблицы, включая обязательное значение `Все` |
| Извлечение строк металла | Профиль, материал, количество, длина, масса, источник и предупреждения |
| Нормализация профилей | Приведение `I25K2`, `TПC200-1.5`, `Лист г/к 8 мм`, `Гнз160х100х6` к технически единому виду |
| Классификация материалов | С245, С255, С345, 09Г2С, Ст3, S350GD, S235JR, ЛСТК, лист, гнутый профиль |
| Мапинг ИНСИ | Exact, normalized, fuzzy, needs_review и not_found без скрытой подмены материала |
| Excel-экспорт | Листы `1.1.1_Выборка металла`, `Мапинг_ИНСИ`, `Несопоставлено`, `Контроль`, `Найденные_таблицы`, `Исходные_строки` |
| Firecrawl / AI | Опциональные усилители, отключены по умолчанию; проект запускается без API-ключей |

## Process Automated

1. Пользователь загружает PDF/Excel проекта и Excel номенклатуры ИНСИ.
2. Система находит потенциальные таблицы и показывает их в интерфейсе.
3. Пользователь фильтрует таблицы по типу и материалу.
4. Сервис извлекает позиции металла в структуру `MetalRow`.
5. Профили нормализуются к единому техническому обозначению.
6. Материалы и группы классифицируются rule-based логикой.
7. Позиции сопоставляются с номенклатурой ИНСИ.
8. Спорные строки получают статус `needs_review`, отсутствующие — `not_found`.
9. Пользователь выгружает Excel в форме ИНСИ.

## Why AI-assisted, Not AI-only

Проект намеренно не построен как “чат с PDF”. Основная ценность — проверяемая детерминированная логика:

- парсинг таблиц;
- нормализация профилей;
- фильтрация;
- классификация материалов;
- мапинг с номенклатурой;
- контроль массы;
- Excel-экспорт.

AI может использоваться только как помощник: объяснить спорную строку, предложить нормализацию или подготовить текстовое пояснение. Он не должен автоматически пересчитывать массу, подменять материал или принимать спорный мапинг.

## Architecture / Modules

| Module | Role |
|--------|------|
| `backend/app/parsers` | Локальные адаптеры PDF/Excel и Firecrawl-заготовка |
| `backend/app/services/profile_normalizer.py` | Нормализация профилей и разных написаний |

| `frontend/src` | React-интерфейс загрузки, фильтров, таблиц, итогов и экспорта |

## Business Impact

- **Сокращение ручного переноса данных**: спецификации и выборки металла собираются в единую ведомость.
- **Снижение риска ошибок**: сомнительные строки не удаляются, а маркируются warning/status.
- **Стандартизация номенклатуры**: разные обозначения профилей приводятся к единому виду перед мапингом.
- **Контроль снабжения**: несопоставленные позиции видны отдельным листом Excel.
- **Прозрачность инженерной логики**: расчеты и итоговые данные выполняются кодом, а не скрытым AI-решением.
