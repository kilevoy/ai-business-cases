# Автоматизация подбора металлокаркаса и ограждающих конструкций

> Веб-приложение для переноса Excel-расчетов подбора колонн, прогонов, ферм и ограждающих конструкций в проверяемый TypeScript-инструмент с инженерной верификацией.

---

## Business Problem

В расчетах промышленных зданий часть инженерной и коммерческой подготовки держится на Excel-файлах: отдельно подбираются колонны, прогоны, фермы, нагрузки и ограждающие конструкции. Такой процесс сложно поддерживать и проверять:

- формулы и справочники распределены между workbook-файлами;
- менеджер или инженер вручную переносит параметры здания между расчетными блоками;
- при изменении города, пролета, шага рам, покрытия или цен нужно пересчитывать связанные разделы;
- ошибки в ручном переносе данных могут влиять на массу, стоимость и спецификацию;
- изменения в расчетной логике трудно контролировать без регрессионных тестов.

Задача проекта — сохранить совместимость с исходными Excel-расчетами, но перевести процесс в единый веб-инструмент с модульной архитектурой и автоматическими проверками.

---

## What Was Done

Создано React/TypeScript-приложение MetalCalc для предварительного подбора элементов здания и формирования сводной спецификации. Расчетные ядра вынесены из интерфейса в доменные модули, а справочные данные генерируются из Excel-источников или проверенных снимков.

| Feature | Description |
|---------|-------------|
| Единый ввод параметров | Пролет, длина, высота, шаг рам, город, снег/ветер, тип кровли и покрытия используются несколькими расчетными блоками |
| Подбор колонн | Расчет крайних, фахверковых и внутренних колонн с автоматическим и ручным выбором профилей |
| Подбор прогонов | Подбор сортового проката и ЛСТК-веток МП350/МП390 с расчетом нагрузок и кандидатов |
| Подбор ферм | Расчет групп элементов фермы с проверками прочности и устойчивости |
| Ограждающие конструкции | Расчет стеновых и кровельных сэндвич-панелей, доборных элементов, крепежа и уплотнителей |
| Сводная спецификация | Итог по колоннам, прогонам, фермам и ограждающим конструкциям в одном интерфейсе |
| Импорт цен | Для ограждающих конструкций предусмотрен импорт цен из PDF-прайса и сохранение переопределений |
| Инженерная верификация | Есть генераторы reference-данных, parity-тесты, unit-тесты, e2e-тесты и команда release-проверки |

## Process Automated

Автоматизирован процесс от исходных параметров здания до расчетной базы для спецификации:

1. Пользователь вводит геометрию здания, город, нагрузки, тип кровли, покрытие и ограничения подбора.
2. Система рассчитывает производные параметры: снеговые и ветровые нагрузки, высоты колонн, шаги, расчетные усилия.
3. Модуль колонн подбирает профили по группам и формирует массу, стоимость и ведомость.
4. Модуль прогонов считает нагрузочную цепочку и предлагает варианты сортового проката или ЛСТК.
5. Модуль ферм использует результаты нагрузок и подбирает группы элементов фермы.
6. Модуль ограждающих считает стеновые и кровельные панели, крепеж, уплотнители и доборные элементы.
7. Сводная вкладка объединяет результаты в общую спецификацию по массе и стоимости.

## Architecture / Modules

| Module | Role |
|--------|------|
| `src/app` | Оболочка приложения, глобальная композиция и метаданные |
| `src/pages/calculator` | Единый экран расчета, панель ввода, вкладки результатов и сводка |
| `src/domain/column` | Подбор колонн, группы колонн, ранжирование профилей и спецификация |
| `src/domain/purlin` | Расчет прогонов, нагрузочная цепочка, сортовой прокат и ЛСТК МП350/МП390 |
| `src/domain/truss` | Расчет фермы, усилия, проверки групп элементов и масса фермы |
| `src/domain/enclosing` | Расчет сэндвич-панелей, доборов, крепежа, уплотнителей и цен |
| `src/shared/excel` | Excel-совместимые числовые helper-функции |
| `scripts` | Извлечение reference-данных из workbook-файлов и генерация acceptance-артефактов |
| `tests/unit` | Unit- и parity-проверки расчетной логики |
| `tests/e2e` | Browser smoke-тесты приложения |

Ключевой архитектурный принцип проекта: интерфейс не хранит бизнес-формулы, расчетные правила вынесены в доменные модули и покрываются тестами.

## Business Impact

- **Снижение ручного переноса данных**: параметры здания вводятся один раз и используются связанными расчетными блоками.
- **Стандартизация подбора**: колонны, прогоны, фермы и ограждающие считаются по единым правилам внутри одного инструмента.
- **Контроль изменений**: reference-генераторы и тесты помогают отслеживать расхождения с исходными Excel-расчетами.
- **Ускорение подготовки расчетной базы**: сводная спецификация собирается автоматически из результатов модулей.
- **Поддерживаемость**: расчетная логика отделена от UI, поэтому новые ветки и справочники можно добавлять без переписывания интерфейса.
- **Прозрачность для инженерной проверки**: в проекте есть документы по parity-стратегии, acceptance-сценариям и архитектуре.
