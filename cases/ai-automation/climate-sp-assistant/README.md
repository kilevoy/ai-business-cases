# Telegram-бот по климатическим данным СП

> N8N-автоматизация, которая превращает статический справочник `climate-sp-atlas` в Telegram-бота: пользователь вводит город, бот возвращает нормативные климатические данные для инженерных расчетов.

---

## Business Problem

Инженерам, менеджерам и специалистам по подготовке КП часто нужно быстро получить исходные климатические параметры по городу: снеговой район, ветровой район, гололед, сейсмичность, глубину промерзания и расчетные температуры.

Веб-атлас уже решает задачу поиска в браузере, но в ежедневной работе удобнее иметь быстрый канал в мессенджере:

- не открывать сайт для каждого уточнения;
- отправлять город из телефона или рабочего чата;
- получать короткую карточку параметров в одном сообщении;
- использовать один и тот же проверенный JSON-справочник без ручного копирования.

---

## What Was Done

Создан Telegram-бот на n8n, который принимает название города, загружает публичный JSON-справочник `climate-sp-atlas`, ищет населенный пункт и отправляет структурированный ответ.

| Feature | Description |
|---------|-------------|
| Telegram input | Пользователь отправляет город обычным сообщением |
| Data source | Бот читает `settlements-climate.json` из GitHub Pages проекта `climate-sp-atlas` |
| City search | Поиск по точному названию, частичному совпадению и региону |
| Multiple matches | Если найдено несколько вариантов, бот показывает список уточнений |
| Help mode | `/start` и `/help` возвращают краткую инструкцию |
| Engineering summary | Ответ содержит снег, ветер, гололед, сейсмику, промерзание и температуры |
| HTML formatting | Сообщение форматируется через Telegram HTML parse mode |
| No AI dependency | Основная логика детерминированная: JSON + поиск + шаблон ответа |

---

## Process Automated

1. Пользователь пишет боту город, например `Иркутск`.
2. Telegram Trigger запускает workflow в n8n.
3. HTTP Request загружает актуальный JSON-справочник климатических данных.
4. Code node нормализует текст запроса и ищет подходящий населенный пункт.
5. Если город найден, workflow формирует инженерную карточку.
6. Telegram node отправляет ответ пользователю в тот же чат.

---

## Architecture / Workflow

| Node | Role |
|------|------|
| `Telegram: City Request` | Принимает входящие сообщения от Telegram-бота |
| `Fetch Climate Atlas Data` | Загружает `settlements-climate.json` |
| `Prepare Telegram Reply` | Нормализует запрос, ищет город, формирует HTML-текст ответа |
| `Telegram: Send Climate Reply` | Отправляет результат пользователю |

Источник данных:

- `https://kilevoy.github.io/climate-sp-atlas/settlements-climate.json`
- 1096 населенных пунктов;
- данные по СП 20.13330, СП 131.13330, СП 22.13330 и СП 14.13330;
- статусы проверки и источники сохраняются в исходном атласе.

## Business Impact

- **Быстрее получение исходных данных**: город можно отправить прямо в Telegram и сразу получить параметры.
- **Единый источник справочных данных**: бот использует тот же JSON, что и веб-атлас.
- **Меньше ручного поиска**: не нужно открывать нормативные таблицы или искать город в нескольких источниках.
- **Удобно для мобильной работы**: параметры можно получить с телефона на объекте, встрече или в переписке.
- **Простое масштабирование**: workflow можно расширить кнопками, выгрузкой PDF, расчетными ссылками или интеграцией с CRM.

---

## Security / Operations

- Telegram token хранится только в n8n credentials.
- В репозитории нет секретов, `.env` или токенов.
- Workflow можно перевыпустить с новым token без изменения логики.
- Основной справочник публичный и не требует API-ключей.
